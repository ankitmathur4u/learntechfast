Big data is traditionally stored and computed using Hadoop technologies which involves managing a lot of underlying in-house infrastructure, upscaling challenges and many other roadblocks. With the emergence of Cloud technologies, it has become easier to bring your focus back to data and business problems, instead of the infrastructure.

 

Google Cloud offers following options-

 

1.     Storage

a.     Batch Data

                                               i. Google Cloud Storage

                                             ii. BigQuery

b.     Real time data

                                               i. Cloud BigTable

                                             ii. Cloud Spanner

2.     Compute/ Data processing

a.     DataProc

b.     DataFlow

3.     Orchestrate

a.     Cloud Composure

4.     Data ingestion

a.     DataFlow

b.     Pub/Sub

c.     Gsutil

d.     Transfer appliance

5.     Data mining and exploration

a.     Cloud Datalab

b.     Cloud Dataprep

6.     Data visualisation

a.     Looker

 

 

Approaches available to transfer data

1.     A public internet- using public API

2.     Direct Peering- using public API

3.     Cloud Interconnect – using private API

 

Drawbacks of Public Internet

1.     Unpredictable network throughput

2.     Limited SLA with internet service provider

3.     Security concerns for sensitive data

 

Direct Peering

1.     Avoids risk of data going on public internet

2.     Limited hops between your network and Google’s network

 

Cloud Interconnect

1.     Avoid risk of data going in public internet and more consistent throughput

2.     Pledged SLA for network and performance

3.     Supports RFC 1918- which makes cloud an extension of your private data centre without public IP address or NAT

 

1 Petabyte of data on 100 Gbps can take 30 hours to transfer
